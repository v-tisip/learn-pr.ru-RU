Microsoft Azure — это постоянно расширяемый набор облачных служб, которые помогают вашей организации решать текущие и будущие бизнес-задачи. Azure предоставляет возможность создания, управления и развертывания приложений в больших глобальных сетях с помощью ваших любимых средств и платформ. В данном разделе представлен подробный обзор ресурсов, доступных в Azure.

### <a name="compute"></a>Службы вычислений

Службы вычислений — это одна из основных причин того, что компании переходят на платформу Azure. Azure предоставляет широкий набор параметров для размещения приложений и служб, в том числе:

- Инфраструктура как услуга (**IaaS**)
- Платформа как услуга (**PaaS**)
- Функции как услуга (**FaaS**)

Предоставляются следующие службы:

|  Имя службы             | Функция службы                                                          |  Тип     |
| -------------             | -------------                                                             | --------- |
| Виртуальные машины          | Виртуальные машины Windows или Linux, размещенные в Azure                                      | IaaS      |
| Служба контейнеров Azure   | Позволяет управлять кластером виртуальных машин для контейнерных служб    | IaaS      |
| Service Fabric            | Платформа распределенных систем. Работает в Azure или локально                | PaaS      |
| Пакетная служба Azure               | Управляемая служба для параллельных и высокопроизводительных вычислительных приложений  | PaaS      |
| Облачные службы            | Управляемая служба для выполнения облачных приложений                            | PaaS      |
| Экземпляры контейнеров Azure | Предоставляет контейнеры без подготовки виртуальной машины или служб более высокого уровня      | FaaS      |
| Функции Azure           | Управляемая служба FaaS                                                      | FaaS      |

### <a name="networking"></a>Сеть

Связывание вычислительных ресурсов и предоставление доступа к приложениям — это ключевая функция сетей Azure. Функции сетей в Azure включают в себя ряд вариантов для соединения внешнего мира со службами и компонентами в глобальных центрах обработки данных Microsoft Azure.

Сетевые средства Azure предлагают следующие возможности:

|  Имя службы             | Функция службы                                                                      |
| -------------             | -------------                                                                         |
| Виртуальная сеть           | Подключение виртуальных машин к входящим подключениям виртуальной частной сети (VPN)                     |
| Балансировщик нагрузки             | Балансировка входящих и исходящих подключений к приложениям или конечным точкам служб         |
| Шлюз приложений       | Оптимизация доставки фермы серверов приложений и одновременное повышение безопасности приложений               |
| VPN-шлюз               | Доступ к виртуальным сетям Azure через высокопроизводительные VPN-шлюзы                   |
| Azure DNS                 | Обеспечивает сверхвысокую скорость ответов DNS и сверхвысокую доступность домена                   |
| Сеть доставки содержимого  | Доставка содержимого по сети с высокой пропускной способностью клиентам по всему миру                                  |
| Защита от атак DDoS Azure     | Защита приложений, размещенных в Azure, от распределенных атак типа "отказ в обслуживании" (DDoS)   |
| Диспетчер трафика           | Распределение сетевого трафика между регионами Azure по всему миру                             |
| ExpressRoute              | Подключение к Azure через выделенные безопасные соединения с высокой пропускной способностью                     |
| Наблюдатель за сетями           | Мониторинг и диагностика неполадок в сети с помощью анализа на основе сценариев                     |
| Брандмауэр Azure            | Реализация брандмауэра с высоким уровнем безопасности и доступности с неограниченной масштабируемостью         |
| Виртуальная глобальная сеть               | Создание единой глобальной сети (WAN), подключенной к локальным и удаленным сайтам           |

### <a name="storage"></a>Хранилище

Azure предоставляет четыре основных типа служб хранилища. Такими службами являются:

- **Хранилище BLOB-объектов** — предоставляет хранилище для очень больших объектов, например видеофайлов или растровых изображений
- **Хранилище файлов Azure** — создает файловые ресурсы, которыми можно управлять как файловым сервером
- **Хранилище очередей Azure** — реализует хранилище для очереди и надежной доставки сообщений между приложениями
- **Хранилище таблиц Azure** — состоит из хранилища NoSLQ, в котором размещены неструктурированные данные, независимые от любой схемы

Все эти службы имеют общие характеристики:

- Надежность и высокая доступность с избыточностью и репликацией.
- Защита посредством автоматического шифрования и управления доступом на основе ролей.
- Масштабируемость с практически неограниченным хранилищем.
- Управляемое обслуживание и решение критических проблем без вашего участия.
- Доступ из любой точки мира по протоколу HTTP или HTTPS.

### <a name="mobile"></a>Мобильные приложения

Azure позволяет разработчикам легко и быстро создавать интересные приложения для iOS, Android и Windows на различных языках и в различных средах разработки. Функции, которые раньше отнимали много времени и повышали риски для проекта, например добавление корпоративного единого входа и подключение к локальным ресурсам, таким как SAP, Oracle, SQL Server и SharePoint, теперь легко использовать.

Другие функции этой службы:

- Синхронизация данных в автономном режиме.
- Подключение к локальным данным.
- Рассылка push-уведомлений.
- Автоматическое масштабирование в соответствии с потребностями бизнеса.

|  Имя службы             | Функция службы                                                                                |
| -------------             | -------------                                                                                   |
| Azure Cosmos DB           | Глобально распределенная база данных, которая поддерживает параметры NoSQL                                       |
| Базы данных SQL Azure        | Полностью управляемая реляционная база данных с автоматическим масштабированием, интегральной аналитикой и надежной системой безопасности    |
| База данных Azure для MySQL  | Полностью управляемая и масштабируемая реляционная база данных MySQL с высокой доступностью и безопасностью        |
| База данных Azure для PostgreSQL   | Полностью управляемая и масштабируемая реляционная база данных PostgreSQL с высокой доступностью и безопасностью   |
| SQL Server на виртуальных машинах         | Размещение корпоративных приложений SQL Server в облаке                                                  |
| Хранилище данных SQL        | Полностью управляемое хранилище данных с интегральной безопасностью на каждом уровне масштабирования без дополнительной платы    |
| Служба миграции баз данных Azure    | Миграция баз данных в облако без изменения кода приложения                            |
| Кэш Redis               | Кэширование часто используемых и статических данных для снижения задержек в данных и приложениях                    |
| База данных Azure для MariaDB      | Полностью управляемая и масштабируемая реляционная база данных MySQL с высокой доступностью и безопасностью        |

### <a name="web"></a>Web

Веб-службы в Azure включают следующие средства:

- Служба приложений — ускоренное создание эффективных облачных приложений для веб-клиентов и мобильных клиентов
- Сеть доставки содержимого — безопасная и надежная доставка содержимого благодаря глобальному охвату аудитории
- Центры уведомлений — отправка push-уведомлений из любой серверной части на любую платформу
- Управление API — безопасная публикация API для разработчиков, партнеров и сотрудников с учетом масштаба
- Поиск Azure — полностью управляемая функция поиска как услуга
- Веб-приложения — быстрое создание и развертывание критически важных веб-приложений с возможностью масштабирования
- Служба Azure SignalR — простое добавление функциональных возможностей в режиме реального времени
